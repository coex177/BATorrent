# Журнал змін

## v3.0.2

### Сполучення з телефоном та WebUI
- Браузерний WebUI **перероблено у стилі десктопної програми** — та сама темна палітра, шрифт Inter, плоскі поверхні, справжній логотип BATorrent (раніше була випадкова кажанча) та нормальна іконка магнету. Тепер це виглядає як один продукт, а не окрема панель управління.
- **Сполучення — одним дотиком, без жодного введення**: згенерований пароль WebUI тепер можна скопіювати, а QR-код містить облікові дані — скануєш телефоном і одразу входиш (без введення IP або пароля), після чого облікові дані зникають з адресного рядка.

### Пошук
- Два нових провайдери: **RuTor** (джерела СНД, без реєстрації, через публічний TorAPI-ретранслятор) і **Torrents-CSV**.
- Результати **відсортовані за кількістю сідів** (найживіші перші), і кожен пошук тепер завершується через 15 с, щоб один мертвий провайдер не підвішував UI.

### Файли та трекери
- **Пріоритет файлів** повернувся: клацніть правою кнопкою на файлі у панелі подробиць, щоб встановити Пропустити / Низький / Нормальний / Високий.
- **Перейменування окремого файлу** всередині торренту (подвійний клік або меню файлу) — окремо від перейменування самого торренту.
- **Видалення трекера** з торренту (✕ на рядку трекера); додавання вже було.
- **Розумна вставка через Ctrl+V** — вставте магнет, 40-символьний info-hash або URL `.torrent` прямо з буфера обміну, і він одразу додається (текстові поля, як і раніше, вставляють текст звично).

### Обкладинки та назви
- Аніме-іменування фансабів (`[Група] Назва - NN`) тепер правильно визначає шоу.
- Розкладки аудіоканалів у назвах (`DDP5.1`, `7.1`, …) прибираються, щоб не засмічувати пошук обкладинок.

### Під капотом
- **Застарілий QWidget-інтерфейс видалено.** QML був єдиним UI починаючи з 3.0.0 (старий код жив за прихованим прапорцем `--legacy`); після підтвердження рівнозначності весь шар QWidget — головне вікно, кожен діалог, менеджер тем — видалено (~13 400 рядків). Чотири відновлені функції вище були можливостями, які бекенд вже підтримував, але QML-порт ніколи не підключив.
- **macOS:** *хеш* пароля WebUI переїхав з keychain до налаштувань програми, тому запуск більше не показує запит до login-keychain на непідписаних збірках. Сам пароль як і раніше живе в keychain.
- Прибирання: ~400 осиротілих рядків перекладу і пакет мертвого коду видалено; внутрішні дублювання усунуто; додано `ARCHITECTURE.md` для контриб'юторів. Юніт/безпека/пам'ять тести та санітайзери ASan/UBSan/TSan залишаються зеленими.

## v3.0.1

### Windows / Linux
- **Відновлено рядок меню** (Файл, Торрент, Налаштування, Довідка — включно з Налаштуваннями, Перевіркою оновлень і Про програму). Він зникав на Windows, бо попередній рядок рендерився лише як глобальне меню macOS; тепер малюється всередині вікна, тоді як macOS зберігає нативне глобальне меню.

### Обкладинки та назви
- Значно покращений розбір назв: початкові префікси з сайтів релізів (`www.foo.com - `, `[ tracker.net ] - `) прибираються, і для епізодів використовується лише назва шоу до SxxExx — тому `www.UIndex.org - Euphoria US S03E08 in God We trust` визначається як Euphoria.
- Плитки сітки та списку більше не показують порожній підпис, поки обкладинка не вирішилась — відображається розібрана назва, потім сира назва торренту. Режим списку тепер збігається зі сіткою.
- Плитки епізодів показують SxxExx, тому кілька епізодів одного шоу можна розрізнити.
- **Виправлення неправильної обкладинки** через контекстне меню: прив'яжіть торрент до правильної назви Фільму / Серіалу / Гри або очистіть через «Без обкладинки». Перевизначення запам'ятовується і ніколи не перезаписується автоматичним збігом.

### Сповіщення
- Події завершення / помилки / аварійного відключення / RSS знову показуються як справжні сповіщення ОС (видимі коли вікно згорнуте), а не лише як внутрішні тости.

### Інше
- Портативне завантаження для Windows тепер називається `BATorrent-windows-x86_64-portable.zip`, щоб інсталятор був очевидним вибором.

## v2.5.2

### Стабільність (на основі аналізу коду qBittorrent)
- Try-catch навколо всього тіла циклу `processAlerts` — один поганий alert більше не крашить програму
- `active_checking=1` — лише один торрент перевіряється одночасно (запобігає OOM на торрентах 96 ГБ+)
- `checking_mem_usage=512` — явний бюджет пам'яті для перевірки частин (8 МБ)
- Інвалідація кешу в `forceRecheck()` — першопричина краша при перевірці 96 ГБ
- `alert_queue_size=1000000` — щедра черга, щоб шторми «диск повний» не скидали alerts мовчки
- Захист від петлі крашів — прапорець `startupInProgress` у QSettings; пропускає дані відновлення при краші під час завантаження
- Обмежена частота емісій `file_error_alert` (1 на 30 с) — «диск повний» більше не генерує сотні сповіщень на секунду
- Авто-пауза всіх завантажень при виявленні повного диска
- Дедублікація помилок на рівні торренту
- Обробники для `fastresume_rejected_alert`, `torrent_checked_alert`, `alerts_dropped_alert`, `storage_moved_failed_alert`

### Розширені налаштування (18 параметрів libtorrent)
- Дисковий I/O: асинхронні потоки, потоки хешування, розмір пулу файлів, пам'ять для перевірки, позначка буфера відправки
- З'єднання: глобальний ліміт, швидкість з'єднань, слоти розблокування, максимум завантажень/з'єднань на торрент
- Алгоритми: choking (фіксовані слоти / на основі швидкості), seed choking (по черзі / найшвидше завантаження / проти личерів)
- Перемикачі: врахування IP-накладних у лімітах швидкості, звільнення LAN-пірів від обмежень

### Автоматизація
- **Запустити після завершення** — зовнішня команда зі змінними шаблону (%N=назва, %D=шлях, %H=хеш, %Z=розмір, %F=файл)
- **Папка спостереження** — авто-додає `.torrent`-файли кожні 10 с, переміщає до `.processed/` після додавання
- **Директорія експорту торрентів** — авто-копіює `.torrent`-файли до резервної папки при додаванні
- **Черга завантажень** з виявленням застряглих торрентів (10 КБ/с протягом 60 с = звільняє слот черги)

### Просунуті можливості
- Режим **суперроздачі** для початкового розповсюдження
- **Примусовий запуск** — обхід ліміту активних завантажень у черзі для одного торренту
- **Ліміти швидкості на рівні торренту** (завантаження + роздача, зберігаються за info-hash)
- **Зупинка після завантаження та максимальний час роздачі на рівні торренту** (перевизначає глобальні значення за замовчуванням)
- **Планувальник пропускної здатності** — альтернативний швидкісний профіль з розкладом по годинах і днях тижня
- **Авто-завершення** — позначає торрент як Завершений після налаштовуваного вікна роздачі

### Полірування
- Скасування видалення через тост + історія нещодавно видалених (останні 50 торрентів, відновлення в один клік)
- Діагностика `diagnoseSlow()` для зависаючих торрентів
- Попередження про малий об'єм диска (менше 1 ГБ)
- Пауза завершених торрентів при помилках файлів (відключений зовнішній диск)
- Опортуністичне збереження даних відновлення при `piece_finished_alert` (з обмеженням 1/хв/торрент)
- Файловий журнал з ротацією (5 МБ/файл, зберігати 3) + діалог переглядача журналу з фільтром рівнів і експортом

- **Дзеркало оновлень Gitee** — альтернативне джерело оновлень для користувачів у Китаї

### Інтерфейс
- Система мульти-тегів (довільні, кілька на торрент)
- Обхід черги через Примусовий запуск
- Історія нещодавно видалених (останні 50, відновлення в один клік)
- Повне резервне копіювання/відновлення налаштувань + даних відновлення
- Вбудоване перейменування (F2)
- Поновлення при зміні мережі через QNetworkInformation
- Спливаюче вікно журналу змін після оновлення версії
- Відображення швидкості в байтах або бітах (перемикається)
- Форматування чисел з урахуванням локалі
- 8 мов UI: EN, PT, ZH, JA, RU, ES, DE, UK
